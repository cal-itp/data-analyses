BART Ridership: BART provides daily reported Daily Boarding and Daily Ridership data from October 1, 2024 to September 30, 2025. The dataset includes day-type classifications such as Holiday, Saturday, Sunday, and Weekday. For modeling purposes, holiday observations were normalized to the standard service day categories of Weekday, Saturday, and Sunday based on their operating characteristics. The ridership fields were converted to numeric values to ensure consistency, and average boardings and alightings were calculated at the stop level by grouping records by dataset, organization, service, stop information, and day type. This process produces representative average ridership values for each stop across the three standard service day categories.


Big Blue Bus: Big Blue Bus ridership data is provided as averages across four different time periods. To create a representative stop-level dataset, boardings and alightings were summed across directions within each stop and weighted by the number of service days in each period. A weighted average was then calculated for each stop to produce representative daily boardings and alightings for weekdays, Saturdays, and Sundays, and a flag was set to indicate that these values are based on reported average daily ridership.


Caltrain ridership data is provided as average ridership per station over multiple time periods, though it is unclear if it represents boardings or total ridership. Holidays are removed, and the number of service days for weekdays, Saturdays, and Sundays is calculated using U.S. federal holidays. A weighted average of ridership is then computed for each station across all periods, using the number of service days as weights. Finally, a flag is added to indicate that the values represent reported average daily ridership, producing a representative stop-level dataset for modeling purposes.

SacRt : Not processed yet. The SacRT stop-level ridership dataset contains three types of records: weekday, daily, and other. Weekday rows have the Saturday and Sunday flags set to 0 and represent average boardings on Monday through Friday only. Daily rows have all day-of-week flags set to 1, meaning the stop operates every day; for these stops, the average_boardings reflects the average across all 7 days, calculated as an average over a three-month period. Importantly, in the daily type, Saturday and Sunday always appear together, either the stop runs both weekend days or neither, so there are no cases where only one weekend day is active. Other rows are exceptions, for example stops that don’t operate on a specific weekday, and need to be handled individually. There is just one case of Other where one of the stop shows 0 on a Tuesday.  I’m thinking that, for modeling purposes, since weekend-specific averages aren’t directly provided, the best approach would be to take stops with daily records that operate on weekends and use any available weekday data to estimate total weekend ridership using S + U = 7 × daily − 5 × weekday.
 Then, using GTFS data, we can allocate that weekend total across Saturday and Sunday based on the number of trips serving each stop. For example, if a stop has 60 trips on Saturday and 40 trips on Sunday, we would assign 60% of the weekend ridership to Saturday and 40% to Sunday.
